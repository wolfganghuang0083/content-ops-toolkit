這個精靈會幫你產生兩個檔：① project.json（一般設定）② .env（密碼，只存你電腦）。之後跟 Claude 說「用 content-ops 幫某客戶跑一輪」，整條內容鏈就會自動跑：想題目 → 排地圖 → 寫文章 → （達標自動發佈）→ 排程 → 內鏈。
需要準備：一個 WordPress 網站（文章要發在那）、一個 GitHub 帳號（裝工具用）。Google 試算表是選配，不想弄也能用。準備好了就按「下一步」。

🔗 開「憑證 Credentials」頁
下載的檔名通常很長，等一下我們會把它改名成 credentials.json。
把剛下載的鑰匙檔放到定位、跑一個指令，瀏覽器會自動跳出來請你登入「要放內容地圖的那個 Google 帳號」並按允許。複製下面整段貼到終端機執行（記得把第二行的下載路徑換成你實際的）：
mkdir -p ~/.config/google mv ~/Downloads/client_secret_*.json ~/.config/google/credentials.json pip3 install google-auth-oauthlib google-api-python-client python3 ~/.claude/skills/content-ops-toolkit/scripts/authorize_sheets.py
看到「✅ 授權完成！token 已存到 …token.json」就成功了。這把鑰匙之後都不用再弄。

它會自動做：
🧬 想出 5 種客人 × 25 個題目 → 🗺️ 排成內容地圖（先做最划算的）→ ✍️ 一篇篇寫稿（含防亂編的事實查核 + 自動配圖）→ 🚦 達標就自動發佈、沒達標自動退草稿 → 📅 其餘排進未來時段 → 🔗 文章間自動內鏈。
關於「自動上架」：排程是用 WordPress 的「預約發佈」，時間到它自己上線，你不用守著。第一次建議設 先看過再發（精靈第⑥步），確認語氣與事實 OK，再切自動發佈。
真正的「自動化內容行銷系統」＝不用你每週記得跑。設一個定時自動跑的排程，它會固定幫你補新文 + 排上架，行事曆永遠不空。
💡 其實光是第一次跑，就已經幫你把接下來好幾週的文章排好、到點自動上架了；這一步是連「定期補新的一批」都自動化，真正一勞永逸。

設好之後，這個系統就會自己：
• 每週自動寫幾篇新文、排進未來時段
• WordPress 到點自動把文章上線
• 新文自動接進內鏈網
• 把狀態寫回內容地圖（哪些寫了、哪天上）
想暫停／改頻率：跟 Claude 說「列出我的排程」「把 X 的內容排程改成兩週一次／暫停」。
⚠️ 定時排程要能讀到你的 WP 密碼才能自動發 —— 在同一台會開著的電腦跑最簡單；或把機密設進排程環境。不想處理也沒關係：第一次跑就已排好幾週，之後手動每月再跑一次 content-ops 補一批 也等於半自動。
你平常只需要：
• 看行事曆：跟 Claude 說「看 X 的內容排程現況」→ 它列出已排/已上/待寫。
• 核可（若選「先看過再發」）：它會給你後台預覽連結，你按發佈。
• 給方向：想主打某主題就說「這個月多寫 OOO 相關的」。
• 信任後切「達標自動發佈」，就幾乎全自動。
成果長相：文章按行事曆一篇篇自己上線、彼此內鏈、SEO 設定齊全、防亂編把關 —— 一台會自己持續產出的內容行銷機器。
🔒 維運小提醒：① 用完不再給程式用，到 WP 把那組應用程式密碼作廢 ② 內容涉及真實數字/客戶名，務必在 FACTS.md 填好（這是防亂編的命脈）③ 文章列表沒即時更新就到 WP 後台清一次快取。
