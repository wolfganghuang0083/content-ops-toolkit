這個精靈會幫你產生兩個檔：① project.json（一般設定）② .env（密碼，只存你電腦）。之後跟 Claude 說「用 content-ops 幫某客戶跑一輪」，整條內容鏈就會自動跑：想題目 → 排地圖 → 寫文章 → （達標自動發佈）→ 排程 → 內鏈。
需要準備：一個 WordPress 網站（文章要發在那）、一個 GitHub 帳號（裝工具用）。Google 試算表是選配，不想弄也能用。準備好了就按「下一步」。
就是給這次的案子取一個英文小名，之後當檔名用。例如客戶是賣寵物鮮食，就打 pet-food；幫自己公司做就打 my-site。只用英文、數字、減號，不要空格。
就是文章要發佈的那個網站首頁網址，從瀏覽器網址列複製貼上即可（要含 https://）。例如 https://www.yourshop.com。下面那格「技術網址」會自動幫你帶好，通常不用改。
就是你登入 WordPress 後台用的帳號名稱（不是 email、不是密碼，是那個使用者名稱）。等一下程式要用它幫你發文章。
這是讓程式幫你發文章的專用鑰匙，跟你平常登入的密碼不一樣（這樣比較安全，隨時可作廢）。
怎麼拿：點下面按鈕到「個人資料」頁 → 捲到最底 「應用程式密碼 / Application Passwords」 → 名稱打 claude-content → 按「新增」→ 會出現一組像 abcd EFGH 1234 ... 的 24 碼 → 馬上複製（只出現一次）。

你的客人主要用什麼語言、在哪個市場？文章會用這個語言寫。例如 台灣繁體中文、US English。
寫好的文章，要 AI 自己發，還是你看過再發？兩個都安全（自動發也有三道檢查把關，沒過會自動退成草稿、不會亂發）。新客戶剛開始建議先「我看過再發」，熟了再切自動。
不用寫得很完整，白話講就好：你賣什麼、賣給誰、有什麼特色、希望客人做什麼（買單／預約／訂閱）。AI 會用這段自己想出 5 種客人、25 個文章題目。

第一次用會要你同意服務條款，照按即可。建完確認左上角專案名稱是你剛建的那個。
在剛剛的專案裡，按下面連結 → 按大大的 「啟用 / Enable」。這是允許程式去讀寫 Google 試算表。
🔗 開「Google Sheets API」→ 按啟用
這步是建立讓程式登入的鑰匙：
1. 點按鈕到「憑證」頁 → 上方 「建立憑證 → OAuth 用戶端 ID」。
2. 應用程式類型選 「桌面應用程式 / Desktop app」 → 建立。
3. （若第一次要你先設定「OAuth 同意畫面」：使用者類型選「外部」，App 名稱隨填，其他一路下一步即可，再回來建用戶端。）
4. 建好後按那一列右邊的 ⬇️ 下載 JSON，會得到一個檔。
🔗 開「憑證 Credentials」頁
下載的檔名通常很長，等一下我們會把它改名成 credentials.json。
把剛下載的鑰匙檔放到定位、跑一個指令，瀏覽器會自動跳出來請你登入「要放內容地圖的那個 Google 帳號」並按允許。複製下面整段貼到終端機執行（記得把第二行的下載路徑換成你實際的）：
mkdir -p ~/.config/google mv ~/Downloads/client_secret_*.json ~/.config/google/credentials.json pip3 install google-auth-oauthlib google-api-python-client python3 ~/.claude/skills/content-ops-toolkit/scripts/authorize_sheets.py
看到「✅ 授權完成！token 已存到 …token.json」就成功了。這把鑰匙之後都不用再弄。
